Classification Example¶

We'll get started using a Classification example. We'll use the pre-loaded sklearn dataset called digits. It consists of several thousand examples of handwritten digits between 0 and 9 inclusive. These are 8x8 images in grayscale, but the data actually comes in a 64 dimension feature vector per labelled example.

from sklearn.model_selection import learning_curve

# Making a custom scorer but not needed for classification
def make_rmse(estimator, X, y):
    y_pred = estimator.predict(X)
    return  np.sqrt(np.mean((y_pred - y)**2))

# Creating a modified plotting function to better handle the custom solver
def lc_plots(estimator, X, y, title, ylims = None, train_size_list = np.linspace(0.1, 1.0, 10), cv_func = 10, scoring_method = 'custom', y_axis_label = None):
    # Handle the customer scorer, or some other function before defaulting to the defaults
    if scoring_method == 'custom':
        scoring_func = make_rmse
    elif scoring_method is not None:
        scoring_func = scoring_method
    else:
        scoring_func = None
        
    # Create Learning Curve values
    num_samples, train_vals, cv_vals = learning_curve(estimator, X, y, 
                                                      train_sizes = np.array([x for x in train_size_list]), 
                                                      cv = cv_func,
                                                      n_jobs = -1,
                                                      scoring = scoring_func)
    # Make values for plots
    train_scores_mean = np.mean(train_vals, axis = 1)
    train_scores_std = np.std(train_vals, axis = 1)
    cv_scores_mean = np.mean(cv_vals, axis = 1)
    cv_scores_std = np.std(cv_vals, axis = 1)
    fig, ax = plt.subplots(figsize = (10, 8))
    if ylims is not None:
        plt.ylim(ylims)
    ax.fill_between(num_samples, train_scores_mean - train_scores_std,
                         train_scores_mean + train_scores_std, alpha=0.1,
                         color="r")
    ax.fill_between(num_samples, cv_scores_mean - cv_scores_std,
                         cv_scores_mean + cv_scores_std, alpha=0.1,
                         color="g")
    ax.plot(num_samples, train_scores_mean, 'o-', color="r",
                 label="Training score")
    ax.plot(num_samples, cv_scores_mean, 'o-', color="g",
                 label="Cross-validation score")
    ax.set_title(title, fontsize = 20)
    ax.yaxis.set_label_text( y_axis_label )
    ax.xaxis.set_label_text( '# of Samples' )
    plt.legend(loc = 'upper left', fontsize = 14)

Confusion Matrix¶

The Confusion Matrix is a simple table that shows the count of items that had predicted label x when the actual label was y (and note that x and y can be equal). The Confusion matrix provides an easy way to visualize more granular details on your classifier that our earlier metrics couldn't illustrate. For instance, we know our f1-scores are good, but it could be that there is a marginal class that we are not predicting well. Analysis of the Confusion Matrix could illustrate this. Let's see how this looks. We'll start by loading the function from sklearn and creating a function to take the default output and converts it into a more readable pandas dataframe:

In [11]:
from sklearn.metrics import confusion_matrix
import pandas as pd

#  This helpful function takes a confusion matrix object and the labels of your data and converts it into a pd.Dataframe with row and column labels
def cm2df(cm, labels):
    df = pd.DataFrame()
    # rows
    for i, row_label in enumerate(labels):
        rowdata = {}
        # columns
        for j, col_label in enumerate(labels): 
            rowdata[col_label] = cm[i,j]
        df = pd.concat([df, pd.DataFrame.from_dict({row_label: rowdata}, orient = 'index')])
    return df[labels]

ROC/AUC¶

The Receiver Operator Characteristic (ROC) and Area Under the Curve (AUC) are a graphical means of analyzing model performance and summary metric (respectively).

The ROC looks at the True Positive Rate (TPR) and False Positive Rate (FPR) of the classifier as the acceptance threshold for the classifier is varied. Remember that many classifiers return probabilities of membership (e.g. Logistic Regression, Neural Networks, Naive Bayes) and many others can have a probability induced (e.g. SVM, kNN, bagging models). We need to find a way to convert the probabilities into a binary or multiclass label. This is an aspect that is often overlooked in most modeling and this can cause poorer performance of the model.

If we focus on the binary case, the default is to just say all probabilities over 50% become "1", otherwise it is "0". But what if the acceptance threshold were say 15%. Would the model be better? What if we varied the acceptance threshold from 0 to 1 with several increments. At each increment, we compute the Confusion Matrix and calculate the TPR and FPR. We can plot the values on respective axes (as we'll see in a bit). These plots tend to have a solid line (the ROC) and a dashed vertical line through from the origin to the upper-right corner. The dashed line (the reference line) is what a random classifier would do (i.e. it gets as many right as it gets wrong for all levels of acceptance). So any curve above this line is performing better than random guessing.

An ideal classifier would go from (0, 0) straight to (0, 100) and then go to (100, 100). This would represent a classifier that has a TPR of 100 and FPR of 0 (it gets everything right and never gets anything wrong), but this usually never happens though you can get close to this. To further help summarize this, we can find the AUC, which is literally just the area under the curve. Our ideal case would have an area of 1 (ideal) while random guessing would give us an AUC of 0.5. The larger the number, the better the classifier.

At this point, the principal issue to note is that this only works on Binary data and we were trying to predict all possible digits from 0 to 9. But not to fear! We can take a multi-class problem and convert this into a series of binary classifications. The two main ways of doing this are using "One-vs-Rest" (OVR) (for each label, you train a model to distinguish that label from everything else pooled together) and "One-vs-Others" (OVO) (you train a classifier for each possible pair of labels). We'll use OVR in this case, using the following procedure: 1) Binarize the response variable which has m possible labels so that it goes from being an n x 1 vector to an n x m matrix. Each column consists of 0's and 1's only. </br> 2) We wrap a 'OneVsRestClassifier' object around our estimator and predict.

Now since we had already fit one model would could have theoretically just used those hyperparameters and hope for the best, but I wanted to not only show you how to integrate this into a GridSearchCV object AND use the ROC-AUC scoring metric (now that our data supports it). # First aggregate all false positive rates
all_fpr = np.unique(np.concatenate([fpr[i] for i in range(np.shape(bin_classification_y_test)[1])]))

# Then interpolate all ROC curves at this points
mean_tpr = np.zeros_like(all_fpr)
for i in range(np.shape(bin_classification_y_test)[1]):
    mean_tpr += np.interp(all_fpr, fpr[i], tpr[i])

# Finally average it and compute AUC
mean_tpr /= np.shape(bin_classification_y_test)[1]

fpr["macro"] = all_fpr
tpr["macro"] = mean_tpr
roc_auc["macro"] = auc(fpr["macro"], tpr["macro"])

# Plot all ROC curves
plt.figure(figsize = (10, 8))
plt.plot(fpr["macro"], tpr["macro"],
         label='macro-average ROC curve (area = {0:0.2f})'
               ''.format(roc_auc["macro"]),
         color='navy', linewidth=4)

plt.plot([0, 1], [0, 1], 'k--')
plt.xlim([0.0, 1.0])
plt.ylim([0.0, 1.05])
plt.xlabel('False Positive Rate')
plt.ylabel('True Positive Rate')
plt.title('Macro Averaging the multiclass ROC''s')
plt.legend(loc="lower right")
plt.show()

Summary¶

This concludes my introductory posts into scikit-learn! Now we know how to use pipelines and GridSearchCV to orchestrate transformations to our data and tune hyperparameters. Once our models are created, we have several tools at our disposal to assess model performance, from Dummy/simple models to build baseline performance, Learning Curves to detect Bias/Variance regimes and task specific measures that can help diagnose any performance issues.
